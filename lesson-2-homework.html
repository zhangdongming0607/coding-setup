第 2 课作业

读代码回答问题

任务 1：找到入口文件

  1. 打开 Demo 项目（就是第一课用的那个聊天助手项目）
  2. 找到前端的入口文件是哪个（提示：先找 index.html，看里面的 <script> 指向谁）
  3. 截图或写出入口文件的路径
  4. 用几句话描述这个入口文件做了什么

任务 2：画出组件树

  1. 从入口文件出发，顺着 import 找到根组件（通常是 App）
  2. 看根组件的 return 部分引用了哪些子组件
  3. 画出项目的组件树，标注每个组件的职责

格式不限——文本缩进、手绘拍照、用任何你顺手的工具都行。参考格式：

App（管家：管数据、管逻辑）
├── SearchBar（交互：用户输入关键词，触发搜索）
├── ProductList（展示：接收商品数据，渲染列表）
│   └── ProductCard（展示：显示单个商品信息）
└── ShoppingCart（展示+交互：显示已选商品，可删除）

任务 3：追踪一条数据流

  1. 挑一个 Demo 项目里的用户交互（比如发送消息、点按钮）
  2. 像课上一样，从用户操作开始，一步一步追踪数据经过了哪些文件、哪些函数
  3. 写出每一步：
第一步：用户在 ______ 组件里点了 ______
第二步：触发了 ______ 文件的 ______ 函数，它做了 ______
第三步：...

一直追到最终页面更新为止。如果中间有发请求给后端，也要写出来。

小技巧：VS Code 里按 Cmd+Shift+F 可以全局搜索关键词，帮你快速定位代码在哪个文件里。

提交方式

跟 AI 说：

1. 帮我拉取最新代码
2. 帮我创建一个新分支，分支名叫 homework/我的名字/lesson-2
3. 我已经把作业写好放到 homework/我的目录/ 里了，帮我提交，说明是"第 2 课作业"
4. 帮我推到 GitHub 上

然后去 GitHub 网页上提 PR：

  1. 打开 coding-setup
  2. 点页面顶部的黄色提示条 "Compare & pull request"
  3. 标题写 第 2 课作业 - 你的名字
  4. 点 Create pull request

注意：提完 PR 就行了，不要点那个绿色的 Merge pull request 按钮，等老师 review。
