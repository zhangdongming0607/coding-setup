第 2 课教案：读代码 — 看懂 AI 写了什么

目标：让学员掌握"读代码"的基本方法——找到入口、顺着数据流走，能在 Demo 项目中从入口文件一路追踪到页面渲染，并认识读代码时会遇到的 JS 核心概念。

课前准备

讲师侧

  • 确认 Demo 项目本地能跑通
  • 在 VS Code 中提前打开 Demo 项目，折叠 node_modules
  • 准备好「面包屑导航」板书（白板/飞书文档），用于现场画数据流
  • 准备一段 AI 生成的代码片段（用于动手环节）
  • 开启屏幕录制工具

学员侧

  • 第一课的 Demo 项目还在本地
  • VS Code 已安装

课程流程

0:00-0:05 开场 & 回顾

  • 衔接上节课，引出"读代码"的动机
  • 核心问题：AI 写的代码跑起来了但不知道它在干嘛

0:05-0:15 方法论：找入口，顺着数据流走

  • 第一步：找入口（从 index.html 的 script 标签找到 main.tsx）
  • 第二步：顺着数据流走（用户操作 → 函数 → 请求 → 状态更新 → 页面刷新）
  • 穿插讲：import 的两种写法（相对路径 vs 第三方库）
  • Figma 类比：node_modules 像 Assets 面板里的团队组件库

0:15-0:35 实战：跟我读一遍 Demo 项目

找入口（0:15-0:20）

  • 打开 index.html，看 script 标签指向 main.tsx
  • 打开 main.tsx，看它把 App 渲染到 root 容器
  • 讲解 import 写法和路径概念

读根组件 App.tsx（0:20-0:28）

  • 状态声明：messages 和 loading（白板类比）
  • useEffect：页面加载时获取欢迎消息
  • 跳到 api.ts 看 fetchWelcome 的实现
  • 用 Console 面板验证数据流（让学员动手）
  • handleSend：发送消息的三步流程
  • JSX/render：组件树结构（管家与员工的类比）

动手加组件（穿插）

  • 让学员创建 MessageCount 组件
  • 在 App 中引入并传入 messages.length
  • 验证状态变化自动更新页面

追完整数据流（0:28-0:35）

  • 追踪"你好"从输入到显示回复的完整路径
  • ChatInput → App → api.ts → 后端（简略）→ api.ts → App → ChatWindow
  • 后端细节留给第 4 课

0:35-0:40 JS 和 React 的关系 + 速查卡

  • JS 是语言，React 是工具（中文 vs 飞书文档类比）
  • 发速查卡到群里（JS 基础 + React 特有）

0:40-0:50 动手环节

四个任务：

  1. 搜索找到欢迎消息在哪个文件
  2. 从 ChatInput 出发追踪"天气"的数据流
  3. MessageBubble 怎么区分用户消息和助手消息
  4. （加分）加一条新关键词回复要改哪里

VS Code 技巧：Cmd+Shift+F 全局搜索

0:50-1:00 答疑 + 作业 + 预告

  • 总结：找入口，顺着数据流走
  • 作业：读自己的项目，找入口、画组件树、追一条数据流
  • 预告第 3 课：React 心智模型

应急预案

情况 应对
学员的 Demo 项目丢了 在线看 GitHub 代码或看讲师屏幕
学员对 JS 语法完全没概念 聚焦"找关键词"和"看注释"，不教语法
学员在动手环节卡住 降低难度，先做任务 1 获得成功经验
追数据流时学员跟丢了 回到板书，先定位当前环节
课程进度快 当场开始做作业
课程进度慢 砍速查卡环节，课后发群里

关键原则

  1. 一个核心方法：找入口，顺着数据流走
  2. 带着问题读：不是逐行翻译
  3. 不要教语法：只需能认出常见写法
  4. 用已知的代码：所有演示用上节课的 Demo 项目
  5. VS Code 搜索是大招：教 Cmd+Shift+F
