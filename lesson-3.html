第 3 课：React 心智模型 — 组件、状态、渲染

从零写一个 Issue 看板，搞懂 React 的三个核心概念：组件、Props、State。

3.1 函数：代码世界的"菜谱"

在学组件之前，先认识一个基本概念——函数。

函数就是一个有名字的操作。你给它取个名字，告诉它做什么事，以后叫这个名字就能重复执行。就像菜谱——写一次，以后照着做就行。

// 定义一个函数（写菜谱）
function sayHello() {
  console.log("你好！");
}

// 调用函数（照着菜谱做）
sayHello();  // Console 打出"你好！"
sayHello();  // 再叫一次，再执行一次

函数还可以接收数据——叫参数。就像菜谱里的"食材"，同一个菜谱，用不同的食材做出不同的菜：

function sayHello(name) {
  console.log("你好，" + name + "！");
}

sayHello("小明");  // "你好，小明！"
sayHello("小红");  // "你好，小红！"

React 的组件也是一个函数，只不过它 return 的不是 console.log，而是页面内容。参数在 React 里有个专门的名字叫 Props。

3.2 组件：一个函数，返回页面内容

组件就是一个函数，return 什么页面就显示什么。

function IssueCard() {
  return (
    <div className="issue-card">
      <span className="label">Bug</span>
      <h3>首页加载速度太慢</h3>
      <p>负责人：小明</p>
    </div>
  );
}

return 里面那些长得像 HTML 的东西叫 JSX——React 用它来描述页面结构。

CSS 怎么对应到组件上

JSX 里写 className="issue-card"，CSS 文件里用 .issue-card { } 给它加样式。这就是对应关系：

  • className = 给元素取个名字
  • CSS 里的 .名字 { } = 给这个名字定义样式（圆角、背景色、阴影……）

Figma 类比：图层名 = className，右侧 Design 面板 = CSS 规则。

注意：JSX 里用 className 而不是 class，因为 class 在 JavaScript 里是保留字（被占用了），所以 React 换了个名字。

3.3 Props：让组件接收不同的数据

一张写死的卡片没什么用——看板上有很多 Issue，标题不同、标签不同、负责人不同。难道每张卡片都写一个新组件？

答案是用 Props。同一个组件，传不同的数据，显示不同的内容：

function IssueCard({ title, label, assignee }) {
  return (
    <div className="issue-card">
      <span className="label">{label}</span>
      <h3>{title}</h3>
      <p>负责人：{assignee}</p>
    </div>
  );
}

// 一个组件，三张不同的卡片
<IssueCard title="首页加载速度太慢" label="Bug" assignee="小明" />
<IssueCard title="支持暗黑模式" label="需求" assignee="小红" />
<IssueCard title="登录页文案优化" label="设计" assignee="大伟" />

Figma 类比：Master Component 加了 text properties（title、label、assignee），每个 Instance 填不同的值。

大括号 {} 是什么意思

  • JSX 里默认写的都是纯文字。比如 <div>你好</div>，页面就原样显示"你好"
  • 大括号的意思是："这里不是纯文字，放一个变量的值进来"
  • 比如 <h3>{title}</h3> → 大括号里的 title 会被替换成实际传入的值
  • Figma 类比：就像文本图层里引用一个 Variable——文本本身是死的，但那个位置被变量的值替换

同样的道理也用在传 Props 上：

  • title="首页加载速度太慢" — 引号 = 写死的文字
  • title={myVariable} — 大括号 = 活的变量

关键规则：Props 从外面传进来，组件自己不能改。

TypeScript 和 interface

你可能注意到有些代码里多了 interface 这个东西：

interface IssueCardProps {
  title: string;      // 标题，必须是文字
  label: string;      // 标签
  assignee: string;   // 负责人
}

function IssueCard({ title, label, assignee }: IssueCardProps) {
  // ...
}

interface 就是一份"数据说明书"——列清楚这个组件需要哪些数据、每个是什么类型。就像点外卖的订单格式：菜名(文字)、数量(数字)、备注(文字)——少填一个字段，系统就报错。

  • 有了它，VS Code 能帮你检查传参对不对——拼错字段名会画红线
  • .tsx = TypeScript + JSX（有类型检查），.jsx = JavaScript + JSX（没有类型检查）
  • 现在大部分项目和 AI 生成的代码都用 .tsx

3.4 State：让组件能"记东西"

Props 是外面传的，组件自己改不了。但实际的看板上，Issue 的状态是会变的——你点一下就能把"进行中"改成"已完成"。这种会变的数据怎么处理？

普通变量为什么不行

function IssueCard() {
  let status = "进行中";
  return (
    <div>
      <p>状态：{status}</p>
      <button onClick={() => {
        status = "已完成";
        console.log(status);  // Console 能看到"已完成"
      }}>标记完成</button>
    </div>
  );
}

点按钮 → 页面没反应！打开 Console 能看到值确实变了，但页面不动。

类比：就像你在私聊里跟同事说"这个需求改了"，但没有更新飞书文档——你知道改了，同事也知道，但其他人打开文档看到的还是旧的。普通变量就是"私聊"，React 看不到。

用 useState——React 看得到的变量

import { useState } from "react";

function IssueCard() {
  const [status, setStatus] = useState("进行中");
  return (
    <div>
      <p>状态：{status}</p>
      <button onClick={() => setStatus("已完成")}>标记完成</button>
      <button onClick={() => setStatus("进行中")}>重新打开</button>
    </div>
  );
}

点按钮 → 状态变了，页面立刻更新！

useState 就是直接改飞书文档——你一改，所有打开这个文档的人（页面上的所有组件）自动看到最新内容。

拆解写法：const [当前值, 修改函数] = useState(初始值)

Props + State 同时存在

真实的 IssueCard 两种数据都有：

// title, label, assignee → Props（外面传的，不会变）
// status → State（用户可以切换）

const IssueCard = ({ label, title, assignee, status, onChangeStatus }) => {
  return (
    <div className="issue-card">
      <span className="label">{label}</span>
      <h3>{title}</h3>
      <p>负责人：{assignee}</p>
      <p>状态：{status}</p>
      <div>
        <button onClick={() => onChangeStatus("待处理")}>待处理</button>
        <button onClick={() => onChangeStatus("进行中")}>进行中</button>
        <button onClick={() => onChangeStatus("已完成")}>已完成</button>
      </div>
    </div>
  );
};

Props vs State：Props 是外面传的（别人决定），State 是自己管的（自己决定）。

3.5 渲染：数据变了，React 帮你重画

每次 State 变化，React 都会走这 5 步：

  1. 用户点了"已完成"按钮
  2. onClick 触发，调用 setStatus("已完成")
  3. React 收到通知："数据变了"
  4. React 重新执行组件函数，算出新的页面内容
  5. React 对比新旧内容，只更新变了的部分（"进行中"变成"已完成"）

Figma 类比：改 Master Component 的属性 → 所有 Instance 自动刷新。React 也是同一个逻辑，只不过它更新的是浏览器里的真实页面。

关键点：React 不会刷新整个页面，只更新变化的部分。

3.6 State 提升：数据该放在谁那里

之前每张卡片自己管自己的 status。但如果 App 想做筛选（只显示"已完成"的），它看不到每张卡片的状态——因为数据不在它手里。

解决方案：把 status 提升到 App 统一管理。卡片要改状态，就通过 onChangeStatus 通知 App。

export default function App() {
  const [issues, setIssues] = useState([
    { label: "Bug", title: "首页加载速度太慢", assignee: "小明", status: "进行中" },
    { label: "Bug", title: "登录功能异常", assignee: "小明", status: "待处理" },
    // ...
  ]);

  return (
    <div className="app">
      <h1>Issue 看板</h1>
      {issues.map((issue, index) => (
        <IssueCard
          label={issue.label}
          title={issue.title}
          assignee={issue.assignee}
          status={issue.status}
          onChangeStatus={(status) => {
            setIssues(() => {
              const newIssues = [...issues];
              newIssues[index] = { ...issue, status: status };
              return newIssues;
            });
          }}
        />
      ))}
    </div>
  );
}

State 提升的规则：

  • 数据只有一个组件用 → State 放在那个组件里
  • 数据要被多个组件共享，或者父组件需要看到 → State 提升到共同的父组件
  • 子组件通过 onXxx Props 通知父组件"我要改数据" → 父组件调 setXxx 更新

3.7 useEffect：页面加载后的自动触发器

State 管的是"数据变了怎么办"。但有时候你需要页面一打开就自动做一些事——比如去后端拿数据。useEffect 就是这个自动触发器。

const handleEffect = () => {
  alert("欢迎来到 issue 看板");
};

useEffect(handleEffect, []);
  • Figma 类比：原型里的 On Load 交互——页面一打开就自动触发
  • [] = 只在第一次加载时跑一次
  • 真实项目里，这里会去后端拿 Issue 列表——而不是把数据写死在代码里

3.8 对比 Demo 项目

回头看第 2 课的 Demo 聊天项目，和我们刚写的 Issue 看板是同一个模式：

Issue 看板                           Demo 聊天项目
─────────                           ──────────
App（State：issues）                 App（State：messages, loading）
└── IssueCard × N                   ├── ChatWindow
    Props：title, label, status         Props：messages, loading
    回调：onChangeStatus                └── MessageBubble
                                    └── ChatInput
                                        回调：onSend
  • App 统一管理 State（issues / messages）
  • 子组件通过 Props 接收数据
  • 子组件通过回调函数（onChangeStatus / onSend）通知 App 修改数据
  • 数据变了 → 页面自动更新

3.9 同一件事的不同写法

课上用 function 写法，但你在别的项目或 AI 生成的代码里会看到其他写法。它们是同一个意思：

定义组件

// 写法一：function 声明
function IssueCard({ title }) {
  return <h3>{title}</h3>;
}

// 写法二：箭头函数（AI 常用）
const IssueCard = ({ title }) => {
  return <h3>{title}</h3>;
};

// 写法三：箭头函数简写（省略 return）
const IssueCard = ({ title }) => <h3>{title}</h3>;

事件处理

// 写法一：内联（简单逻辑）
<button onClick={() => setStatus("已完成")}>标记完成</button>

// 写法二：单独定义函数（复杂逻辑）
const handleComplete = () => {
  setStatus("已完成");
  console.log("done");
};
<button onClick={handleComplete}>标记完成</button>

TypeScript 类型

// 不用 TypeScript（.jsx）——不写类型也能跑
function IssueCard({ title, label }) { ... }

// 用 TypeScript（.tsx）——编辑器帮你检查
interface IssueCardProps {
  title: string;
  label: string;
  priority?: number;   // 加 ? 表示可以不传
}
function IssueCard({ title, label }: IssueCardProps) { ... }

export

// export default — 一个文件一个，导入时名字随便起
export default function App() { ... }
// import App from "./App"

// 具名 export — 一个文件可以多个，导入时用大括号
export function IssueCard() { ... }
// import { IssueCard } from "./components/issue-card"

3.10 总结

概念 一句话 Figma 类比
组件 一个函数，return 页面内容。定义一次，用多次 Master Component
Props 外面传进来的参数，组件自己不能改 Component Properties
State 组件内部的数据源，改了页面自动刷新（必须用 setXxx） —
useEffect 页面加载后自动执行的代码 On Load 交互
State 提升 多个组件共享数据时，把 State 放到共同的父组件 —
