第 2 课：读代码 — 看懂 AI 写了什么

掌握读代码的核心方法——找到入口、顺着数据流走，从入口文件一路追踪到页面渲染。

在浏览器中打开 Demo

2.1 读代码的方法：找入口，顺着数据流走

用 AI 生成代码之后，你最常遇到的情况大概是：跑起来了，但不知道它在干嘛；出了问题，不知道从哪里找。

读代码的目标不是逐行理解每个字母，而是能定位——知道问题出在哪个文件、哪个环节。方法很简单，就两步：

  1. 找入口。 每个程序都有一个"第一行被执行的代码"，这就是入口。找到它，你就有了起点。
  2. 顺着数据流走。 从入口开始，追踪数据是怎么流动的——用户操作产生了什么数据？数据传给了谁？最终怎么变成了页面上的内容？

这两步适用于任何项目，不管是 React、Vue 还是后端代码。

数据流的基本模型长这样：

用户操作 → 事件处理函数 → 修改数据（状态）→ 页面更新
            ↘ 发请求给后端 → 后端返回数据 ↗

你读代码的时候，就是在这条线上找你关心的那个环节。接下来我们用上节课的 Demo 项目，实际走一遍。

2.2 第一步：找入口文件

打开 Demo 项目的前端代码（client/src/ 文件夹），你会看到好几个文件。从哪个开始看？

记住一个规律：前端项目的入口通常叫 main、index 或 app。在这个项目里，入口就是 main.tsx。

打开 client/src/main.tsx：

/**
 * 前端启动文件 — 整个前端从这里开始
 *
 * 它做的事情很简单：把 App 组件渲染到 HTML 里的 #root 元素中
 */

import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { App } from "./App";
import "./styles.css";

createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <App />
  </StrictMode>
);

这个文件就做了三件事：

  1. 在 HTML 里找到一个叫 root 的容器
  2. 把 App 这个组件塞进去
  3. 页面就出来了

所以现在你知道了——所有的内容都从 App 开始。下一步，去看 App。

关键概念

  • import：意思是"从别的文件里拿东西来用"。就像你在文档里说"参见附件 A"——代码用 import 来引用其他文件里的功能。
  • 组件（Component）：<App /> 看起来像 HTML 标签，但它其实是我们自己写的一个组件。你可以把组件理解为一个可复用的页面零件——按钮是一个零件，输入框是一个零件，整个聊天窗口也是一个零件。

2.3 第二步：读根组件 App

打开 client/src/App.tsx，我们分块来看。下面是这个文件的完整代码：

/**
 * App — 根组件，管理整个页面的状态和布局
 */

import { useEffect, useState } from "react";
import { ChatWindow } from "./components/ChatWindow";
import { ChatInput } from "./components/ChatInput";
import { fetchWelcome, sendMessage } from "./api";
import type { Message } from "./api";

export function App() {
  // 消息列表（状态）——每次更新，页面会自动重新渲染
  const [messages, setMessages] = useState<Message[]>([]);
  // 是否正在等待回复
  const [loading, setLoading] = useState(false);

  // 页面首次加载时，获取欢迎消息
  useEffect(() => {
    fetchWelcome().then((welcomeText) => {
      setMessages([
        {
          role: "assistant",
          content: welcomeText,
          timestamp: new Date().toISOString(),
        },
      ]);
    });
  }, []);

  // 用户发送消息
  const handleSend = async (text: string) => {
    // 1. 先把用户消息加到列表里
    const userMsg: Message = {
      role: "user",
      content: text,
      timestamp: new Date().toISOString(),
    };
    setMessages((prev) => [...prev, userMsg]);
    setLoading(true);

    try {
      // 2. 发请求给后端，等回复
      const { reply, timestamp } = await sendMessage(text);

      // 3. 把回复加到列表里
      const assistantMsg: Message = {
        role: "assistant",
        content: reply,
        timestamp,
      };
      setMessages((prev) => [...prev, assistantMsg]);
    } catch (err) {
      // 请求失败时，把错误信息显示在聊天里
      const errorMsg: Message = {
        role: "assistant",
        content: `⚠️ ${err instanceof Error ? err.message : "请求失败"}`,
        timestamp: new Date().toISOString(),
      };
      setMessages((prev) => [...prev, errorMsg]);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="app">
      <header className="app-header">
        <h1>Demo 聊天助手</h1>
      </header>
      <ChatWindow messages={messages} loading={loading} />
      <ChatInput onSend={handleSend} disabled={loading} />
    </div>
  );
}

代码不短，不用怕。我们一块一块看。

（1）状态声明

const [messages, setMessages] = useState<Message[]>([]);
const [loading, setLoading] = useState(false);

这两行是这个组件的状态（state）。你可以把状态想象成一个白板上的数据——页面上要显示什么、现在是什么情况，都写在这个白板上：

  • messages：消息列表，一开始是空的 []
  • loading：正在等回复吗？一开始是 false（没有在等）

关键特性：状态一变，页面自动刷新。你不需要手动去改页面，只要改数据，React 会帮你更新页面。

（2）useEffect — 页面加载时获取欢迎消息

useEffect(() => {
  fetchWelcome().then((welcomeText) => {
    setMessages([
      {
        role: "assistant",
        content: welcomeText,
        timestamp: new Date().toISOString(),
      },
    ]);
  });
}, []);

这段代码的意思是：页面第一次打开的时候，去后端拿欢迎消息，然后放到消息列表里。

fetchWelcome() 就是上节课你在 Network 面板里看到的那个 /api/welcome 请求。代码写在这里，请求出现在 Network 面板——两边对上了。

（3）handleSend — 用户发送消息

const handleSend = async (text: string) => {
  // 1. 先把用户消息加到列表里
  const userMsg: Message = { role: "user", content: text, ... };
  setMessages((prev) => [...prev, userMsg]);
  setLoading(true);

  try {
    // 2. 发请求给后端，等回复
    const { reply, timestamp } = await sendMessage(text);

    // 3. 把回复加到列表里
    const assistantMsg: Message = { role: "assistant", content: reply, ... };
    setMessages((prev) => [...prev, assistantMsg]);
  } catch (err) {
    // 请求失败时，显示错误信息
  } finally {
    setLoading(false);
  }
};

这就是你点"发送"之后发生的事：

  1. 把你输入的文字加到消息列表里（页面上立刻多一条消息气泡）
  2. 发请求给后端（Network 面板里会出现一个 chat 请求）
  3. 收到回复后，把回复也加到列表里（页面上又多一条气泡）

整个流程就是在改 messages 这个状态。状态变了，页面就自动更新了。

（4）return 部分 — JSX 描述页面结构

return (
  <div className="app">
    <header className="app-header">
      <h1>Demo 聊天助手</h1>
    </header>
    <ChatWindow messages={messages} loading={loading} />
    <ChatInput onSend={handleSend} disabled={loading} />
  </div>
);

最后这部分描述页面长什么样。看起来像 HTML 对吧？这叫 JSX——React 用它来描述页面结构。注意两个自定义组件：

  • ChatWindow：显示消息列表。它接收 messages 数据——App 把自己"白板"上的数据分享给了 ChatWindow。
  • ChatInput：输入框。onSend={handleSend} 的意思是"用户点发送的时候，调用 handleSend 这个函数"。

这就是 React 的组件结构：大组件把数据传给小组件，小组件把用户操作传回给大组件。

2.4 组件树：大组件套小组件

从 App 出发，整个前端页面的组件结构是这样的：

App（管家：管数据、管请求）
├── ChatWindow（展示：接收 messages，画出所有气泡）
│   └── MessageBubble（展示：画一条消息）
└── ChatInput（交互：用户输入，触发 onSend）

我们来看看每个子组件长什么样。

ChatWindow — 聊天窗口

import { useEffect, useRef } from "react";
import { MessageBubble } from "./MessageBubble";
import type { Message } from "../api";

interface ChatWindowProps {
  messages: Message[];
  loading: boolean;
}

export function ChatWindow({ messages, loading }: ChatWindowProps) {
  const bottomRef = useRef<HTMLDivElement>(null);

  // 新消息出现时，自动滚到底部
  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [messages]);

  return (
    <div className="chat-window">
      {messages.map((msg, index) => (
        <MessageBubble key={index} message={msg} />
      ))}
      {loading && (
        <div className="typing-indicator">
          <span></span>
          <span></span>
          <span></span>
        </div>
      )}
      <div ref={bottomRef} />
    </div>
  );
}

ChatWindow 接收 messages 数组，用 .map() 把每条消息变成一个 MessageBubble 组件。如果 loading 为 true，还会显示一个"正在输入"的动画。

MessageBubble — 单条消息气泡

import type { Message } from "../api";

interface MessageBubbleProps {
  message: Message;
}

export function MessageBubble({ message }: MessageBubbleProps) {
  const isUser = message.role === "user";

  return (
    <div className={`message-row ${isUser ? "message-row-user" : "message-row-assistant"}`}>
      {!isUser && <div className="avatar avatar-assistant">AI</div>}
      <div className={`bubble ${isUser ? "bubble-user" : "bubble-assistant"}`}>
        {message.content}
      </div>
      {isUser && <div className="avatar avatar-user">我</div>}
    </div>
  );
}

MessageBubble 通过 message.role === "user" 来判断这条消息是谁发的。用户的消息用 bubble-user 样式靠右显示，助手的消息用 bubble-assistant 样式靠左显示。

ChatInput — 输入框和发送按钮

import { useState } from "react";

interface ChatInputProps {
  onSend: (text: string) => void;
  disabled: boolean;
}

export function ChatInput({ onSend, disabled }: ChatInputProps) {
  const [text, setText] = useState("");

  const handleSubmit = () => {
    const trimmed = text.trim();
    if (!trimmed || disabled) return;

    onSend(trimmed);
    setText("");
  };

  const handleKeyDown = (e: React.KeyboardEvent) => {
    if (e.key === "Enter" && !e.shiftKey) {
      e.preventDefault();
      handleSubmit();
    }
  };

  return (
    <div className="chat-input-bar">
      <input
        className="chat-input"
        type="text"
        placeholder="输入消息..."
        value={text}
        onChange={(e) => setText(e.target.value)}
        onKeyDown={handleKeyDown}
        disabled={disabled}
      />
      <button
        className="send-button"
        onClick={handleSubmit}
        disabled={disabled || !text.trim()}
      >
        发送
      </button>
    </div>
  );
}

ChatInput 有自己的状态 text 来记录用户正在输入的内容。用户点发送（或按 Enter），它会调用从 App 传下来的 onSend 函数，把文字传上去，然后清空输入框。

2.5 追踪一条完整的数据流

现在让我们把所有文件串起来——追踪一个完整的流程：用户发一条"你好"，到页面显示回复，中间经过了哪些文件、哪些函数。

1. 用户在输入框打字、点发送
   → ChatInput.tsx：handleSubmit() 被调用
   → onSend(trimmed) 把文字传给 App

2. App 收到文字
   → App.tsx：handleSend(text) 被调用
   → 先 setMessages 把用户消息加到列表（页面立刻更新）
   → 调 sendMessage(text) 发请求

3. 发请求给后端
   → api.ts：sendMessage() 用 fetch 发 POST 请求到 /api/chat

4. 后端收到请求
   → server/index.ts：app.post("/api/chat", ...) 处理请求
   → 调 getReply(message) 生成回复
   → 返回 JSON 给前端

5. 前端收到回复
   → api.ts：sendMessage() 返回 { reply, timestamp }
   → App.tsx：handleSend() 把回复加到 messages
   → 页面自动更新，ChatWindow 多了一条气泡

6 个文件串起来了：ChatInput → App → api.ts → server/index.ts → api.ts → App → ChatWindow。

我们来看看中间两个关键文件的代码。

api.ts — 前端跟后端通信

/** 消息的数据结构 */
export interface Message {
  role: "user" | "assistant";
  content: string;
  timestamp: string;
}

/** 获取欢迎消息：GET /api/welcome */
export async function fetchWelcome(): Promise<string> {
  const response = await fetch("/api/welcome");
  const data = (await response.json()) as { message: string };
  return data.message;
}

/** 发送消息并获取回复：POST /api/chat */
export async function sendMessage(message: string): Promise<{ reply: string; timestamp: string }> {
  const response = await fetch("/api/chat", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ message }),
  });

  if (!response.ok) {
    const errorData = (await response.json().catch(() => ({}))) as { error?: string };
    const errorText = errorData.error ?? `请求失败 (${response.status})`;
    throw new Error(errorText);
  }

  const data = (await response.json()) as { reply: string; timestamp: string };
  return data;
}

sendMessage() 用 fetch 发了一个 POST 请求到 /api/chat，把用户的消息放在请求体里。后端返回的 JSON 里有 reply 和 timestamp。

server/index.ts — 后端处理请求

app.post("/api/chat", (req, res) => {
  const { message } = req.body as { message: string };

  // ... 错误处理（省略）...

  const reply = getReply(message);        // 根据关键词匹配回复
  const now = new Date().toISOString();

  chatHistory.push({ role: "user", content: message, timestamp: now });
  chatHistory.push({ role: "assistant", content: reply, timestamp: now });

  setTimeout(() => {
    res.json({ reply, timestamp: now });  // 返回 JSON 给前端
  }, 500);
});

后端从 req.body 里拿到前端发来的消息，调 getReply() 生成回复，然后通过 res.json() 返回给前端。

replies.ts — 回复规则

getReply() 的逻辑在 server/replies.ts 里。它根据用户消息中的关键词匹配回复：

const rules: ReplyRule[] = [
  {
    keywords: ["你好", "hi", "hello", "嗨"],
    reply: "你好呀！很高兴见到你，有什么想聊的吗？",
  },
  {
    keywords: ["天气"],
    reply: "我没有联网功能，查不了天气哦。不过你可以打开 DevTools 的 Network 面板，看看我们之间的请求长什么样！",
  },
  // ... 更多规则 ...
];

export function getReply(userMessage: string): string {
  const msg = userMessage.toLowerCase();
  for (const rule of rules) {
    if (rule.keywords.some((kw) => msg.includes(kw))) {
      return rule.reply;
    }
  }
  // 没匹配到关键词，随机选一条通用回复
  return genericReplies[Math.floor(Math.random() * genericReplies.length)];
}

以后遇到 bug，你就沿着这条数据流找——请求发出去了吗？看 api.ts 和 Network 面板。数据返回了吗？看 server。页面没更新？看 App 的状态有没有变。

2.6 JS / React 速查卡

读代码时你可能遇到很多不认识的写法，这很正常。下面是最常见的整理成速查表，遇到了翻一下就行。

JavaScript 基础

写法 含义
import ... from "..." 从别的文件引入功能，像"参见附件"
export function Xxx() 把这个函数公开，让别的文件能 import
const x = 值 声明一个变量（一个有名字的盒子）
函数名(参数) 调用函数（让它执行）
async / await 等待一个需要时间的操作（比如网络请求）
.then(callback) 另一种等待方式，"等结果出来后执行 callback"
interface Xxx { ... } 定义数据的结构（有哪些字段、什么类型）

React 特有

写法 含义
<组件名 属性={值} /> 使用一个组件，给它传数据（叫 props）
useState(初始值) 声明一个状态，返回 [当前值, 修改函数]
useEffect(() => {}, []) 页面加载后执行一次的代码
{条件 && <元素>} 条件为真时才显示这个元素
{数组.map(item => ...)} 把数组里的每一项变成一个页面元素
useRef 拿到一个 DOM 元素的引用，可以直接操作它

不需要背。读代码的时候认不出来就来查这张表，用多了自然就记住了。

2.8 本课小结 + 课后作业

小结

这节课的核心就一个方法：找入口，顺着数据流走。

  1. 找到入口文件（通常叫 main / index / app）
  2. 看入口引用了谁（看 import）
  3. 带着具体的问题，沿着数据流追踪：事件 → 函数 → 请求 → 响应 → 状态更新 → 页面刷新

你不需要一行一行全部读完。读代码是为了回答一个具体的问题——"这个按钮点了之后调了哪个接口？""这段文字是从哪里来的？"有了问题，你才知道该追哪条线。
